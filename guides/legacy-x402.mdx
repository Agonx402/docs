---
title: "Legacy x402 Support"
description: "Accept payments from standard x402 buyers and pay standard x402 merchants — no Agon registration required on the other side."
---

Agon supports two legacy x402 modes that allow interoperability with the broader x402 ecosystem. You can accept payments from non-Agon buyers (Case 3) or pay non-Agon merchants (Case 2).

<Info>
  **Legacy x402 is always enabled.** Every Agon merchant automatically accepts standard x402 buyers — no extra configuration needed. Agon co-signs and broadcasts the transaction as the x402 facilitator, charging a tiny **$0.001 facilitation fee** from your Agon balance.
</Info>

## The Four Payment Cases

| Case | Buyer | Merchant | Mechanism |
|------|-------|----------|-----------|
| 1 | Agon-native | Agon-native | Off-chain settlement (fastest, default) |
| **2** | **Agon-native** | **Standard x402** | Agon acts as buyer proxy, sends `PAYMENT-SIGNATURE` |
| **3** | **Standard x402** | **Agon-native** | Agon acts as x402 facilitator, sponsors tx |
| 4 | Non-Agon | Non-Agon | Reject (merchant has no Agon balance to pay fee) |

---

## Case 3: Accept Standard x402 Buyers

When a buyer without an Agon account wants to pay your Agon-powered API, enable `legacyX402Enabled` in your platform config.

### What happens

1. An x402 buyer requests your endpoint (no `X-AGON-TOKEN`)
2. Your merchant SDK responds with `402 + PAYMENT-REQUIRED` (base64 `PaymentRequirements`)
3. The buyer creates a partial Solana `transferChecked` transaction (signed but **not broadcast**)
4. The buyer retries with the `PAYMENT-SIGNATURE` header containing the base64 tx
5. Your SDK calls Agon's `POST /platform/sponsor-tx`
6. Agon validates the tx, co-signs as `feePayer`, broadcasts, and confirms
7. Your endpoint receives a `200 + PAYMENT-RESPONSE` header with the tx signature

```typescript
import { AgonPlatformCore } from "@agonx402/platform";
import { AGON_HEADERS } from "@agonx402/types";

const core = new AgonPlatformCore({
  agonUrl: "https://api.agonx402.com",
  platformKey: process.env.AGON_PLATFORM_KEY!,
  pricing: 1000, // $0.001 USDC
  // Optional: direct legacy payments to a specific wallet
  // walletAddress: "YourSolanaWalletAddressBase58",
});
```

<Warning>
  **Legacy buyers must hold on-chain USDC.** They are NOT using Agon's custodial balance. Make sure your documentation tells legacy buyers to fund a Solana wallet with USDC before calling your API.
</Warning>

<Warning>
  **Merchant fee.** Each legacy x402 transaction deducts a **$0.001 facilitation fee** from your Agon balance. Ensure your platform account stays funded.
</Warning>

### Middleware example (Express)

```typescript
import express from "express";

const app = express();

app.use("/api", async (req, res, next) => {
  const price = await core.calculatePrice(req);
  const resource = `${req.protocol}://${req.get("host")}${req.originalUrl}`;

  // Standard x402 buyer: PAYMENT-SIGNATURE header present
  const paymentSignature = core.extractPaymentSignature(req.headers);
  if (paymentSignature) {
    const paymentRequiredHeader = await core.buildLegacyPaymentRequiredHeader(price, resource);
    const result = await core.sponsorLegacyTx(paymentSignature, paymentRequiredHeader, price);
    res.setHeader(AGON_HEADERS.PAYMENT_RESPONSE, Buffer.from(JSON.stringify(result)).toString("base64"));
    return next(); // serve the resource
  }

  // Agon-native buyer: X-AGON-TOKEN header present
  const consumerToken = core.extractConsumerToken(req.headers);
  if (!consumerToken) {
    const { status, body } = core.buildPaymentRequiredResponse(price);
    // Also emit the x402 PAYMENT-REQUIRED header for x402 buyers
    const legacyHeader = await core.buildLegacyPaymentRequiredHeader(price, resource);
    res.setHeader(AGON_HEADERS.PAYMENT_REQUIRED, legacyHeader);
    return res.status(status).json(body);
  }

  // ...continue Agon-native authorize/consume flow
});
```

## Case 2: Pay Standard x402 Merchants

When an Agon consumer wants to call a merchant that is **not** on Agon, Agon's proxy acts as a standard x402 buyer.

### What happens

1. Consumer's `agon.fetch()` hits the non-Agon merchant
2. Merchant returns `402 + PAYMENT-REQUIRED`
3. Consumer's SDK calls `POST /proxy` on Agon's backend with the merchant URL and `PAYMENT-REQUIRED` value
4. Agon checks spending controls, reserves funds from the consumer's balance
5. Agon constructs a partial `transferChecked` tx from Agon's custodial wallet, partially signs it, and sends `PAYMENT-SIGNATURE` to the merchant
6. The merchant's configured facilitator broadcasts the tx and pays SOL gas
7. Agon settles the consumer's ledger on receipt of merchant's `200` response

### Consumer code

```typescript
import { AgonClient } from "@agonx402/client";

const agon = new AgonClient({
  baseUrl: "https://api.agonx402.com",
  apiKey: process.env.AGON_API_KEY!,
});

// Transparent — SDK handles the 402/proxy automatically
const res = await agon.fetch("https://any-x402-merchant.com/api/data");
```

No extra configuration is required — the consumer SDK handles this automatically.

## Demo

```bash
# Terminal 1: Start the legacy platform server (accepts both buyer types)
cd packages/demo
npm run platform-legacy

# Terminal 2: Run a standard x402 buyer (holds on-chain USDC)
BUYER_SECRET_KEY=<base58-keypair> npm run legacy-x402-buyer
```
