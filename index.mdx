Agon is the **payment infrastructure that makes x402 scale**. The HTTP 402 "Payment Required" protocol is a powerful primitive for monetizing APIs — but it has a fundamental problem: every API call requires an on-chain transaction. At scale, that means millions of blockchain fees and seconds of RPC latency on every request.

Agon solves this by **decoupling authorization from settlement**. Consumers deposit USDC once. API calls are authorized instantly off-chain via the Agon ledger. Agon settles all on-chain work in a single, efficient transaction per merchant.

<Tip>
  **Zero on-chain transactions on the hot path.** Every API call is authorized via an off-chain ledger lookup. Solana is only touched at deposit, withdrawal, and settlement time.
</Tip>

<Warning>
  **Agon is currently in open beta on devnet.** Sign up at [agon.so](https://agon.so) to get started.
</Warning>

## Who is Agon for?

**🧑‍💻 Developers & Apps**

You want to pay for AI services, data APIs, or compute resources programmatically. Deposit USDC once and your app automatically pays for every call — no wallets, no gas, no RPC latency.

**🤖 AI Agents**

You're building an autonomous agent that needs to purchase API access at runtime. Agon supports native Ed25519 wallet authentication — no browser, no OAuth, no human in the loop. Works with agentic wallets like Coinbase CDP so your agent never needs to manage raw keypairs.

**🏗️ API Merchants**

You want to monetize your API with USDC. Embed the Platform SDK in one middleware and start earning. Agon handles authorization, fraud prevention, and settlement — triggered by you when you're ready to settle.

## x402 at Scale: The Numbers

| Volume | Direct x402 Cost | Agon Cost |
|---|---|---|
| 100 calls → 1 merchant | 100 transactions | 1 settlement tx |
| 1M calls → 50 merchants | 1,000,000 transactions | 3 transactions |
| 1B calls → 50 merchants | 1,000,000,000 transactions | **still 3 transactions** |
