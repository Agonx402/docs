## POST /account/register

Register a new consumer account (web dashboard via Privy).

**Auth:** None (Privy JWT handled by dashboard)

**Request:**
```json
{ "wallet_address": "7xKXtg2NRtu6GKHFjMYHNkr1q5t9mQ..." }
```

**Response `201`:**
```json
{
  "account_id": "uuid",
  "wallet_address": "7xKXtg...",
  "deposit_address": "3mNbz...",
  "available_balance": 0,
  "reserved_balance": 0,
  "balance": 0,
  "currency": "USDC",
  "api_key": "ag_xxxxx",
  "created_at": "2026-02-28T00:00:00Z"
}
```

<Warning>
  `api_key` is returned **once** and never again. Store it immediately in a secret manager or `.env` file.
</Warning>

## POST /account/register/agent

Register an AI agent via Ed25519 wallet signature — no Privy, no browser.

**Auth:** None (signature is the proof of identity)

**Request:**
```json
{
  "wallet_address": "7xKXtg...",
  "signature": "<base58_sig_of_agon:register:timestamp>",
  "timestamp": 1709078400000,
  "beta_token": "A3K7F2M9P1X8"
}
```

| Field | Required | Description |
|---|---|---|
| `wallet_address` | Yes | Base58 Solana public key |
| `signature` | Yes | Ed25519 signature of `agon:register:<timestamp>` using the account keypair |
| `timestamp` | Yes | Unix milliseconds. Must be within ±5 min of server time (replay protection for registration). |
| `beta_token` | Beta only | Single-use invite code. Omit on open mainnet. |

**Response `201`:** Same shape as `/account/register`.

**Error `409 account_already_exists`:** This wallet is already registered.

<Note>
  On a `409`, you have two options:
  - **Use your stored `api_key`** to authenticate subsequent requests via `X-AGON-KEY`
  - **Authenticate via Ed25519 wallet signature** using `X-AGON-WALLET` + `X-AGON-SIGNATURE` + `X-AGON-TIMESTAMP` headers — no API key required
</Note>

  "account_id": "uuid",

    "proxy_allowed_domains": null

## POST /account/create-token

Creates a short-lived consumer token for use with merchant APIs.

**Auth:** `X-AGON-KEY: ag_xxx` or Ed25519 wallet signature headers

**Request:**
```json
{
  "ttl": 60,
  "max_amount": 5000,
  "budget": 50000
}
```

| Field | Default | Max | Description |
|---|---|---|---|
| `ttl` | 60 | 300 | Token validity in seconds |
| `max_amount` | unlimited | — | Max USDC units for a single request via this token |
| `budget` | _(omit)_ | — | Enables multi-use mode. Hard cap on aggregate spend. |

**Response `200`:**
```json
{
  "token": "eyJ...",
  "expires_in": 60,
  "max_amount": 5000,
  "budget": 50000
}
```

<Note>
  Both `X-AGON-KEY` (API key) and Ed25519 wallet signature headers are supported for authentication. See the [Authentication guide](/guides/authentication) for details.
</Note>

## POST /account/deposit

Manual deposit confirmation. Primary path is automatic via Helius webhook — use this as a fallback only.

**Auth:** `X-AGON-KEY: ag_xxx`  **Idempotent on:** `tx_signature`

```json
{ "tx_signature": "5Ht8...xyz" }
```

**Response `200`:**
```json
{ "credited": true, "amount": 5000000, "already_credited": false }
```

---

## POST /account/withdraw

Withdraw USDC to the account's owner wallet on-chain.

**Auth:** `X-AGON-KEY: ag_xxx`

```json
{ "amount": 5000000 }
```

**Response `200`:**
```json
{
  "tx_signature": "3dam...",
  "withdrawn_amount": 5000000,
  "available_balance": 4750000,
  "currency": "USDC"
}
```

**Error `400 withdrawal_exceeds_available`:** Requested amount exceeds `available_balance`.
