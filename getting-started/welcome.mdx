Agon is the **payment infrastructure that makes x402 scale**. The HTTP 402 "Payment Required" protocol is a powerful primitive for monetizing APIs — but it has a fundamental problem: every API call requires an on-chain transaction. At scale, that means millions of blockchain fees and seconds of RPC latency on every request.

Agon solves this by **decoupling authorization from settlement**. Consumers deposit USDC once. API calls are authorized instantly off-chain via the Agon ledger. Agon settles all on-chain work in a single, efficient transaction per merchant.

<Note>
  Agon's on-chain transaction count is bounded by the **number of merchants**, not the number of API calls. The efficiency advantage compounds as your volume grows.
</Note>

## The Agon Solution

Agon is a **custodial payment aggregator** that operates between consumers and merchants:

1. **Consumers deposit USDC** into their Agon account once.
2. **Every API call is authorized off-chain** — an instant off-chain ledger reservation against the consumer's balance.
3. **Merchants accept or deny** the reservation in milliseconds, then serve the response.
4. **Merchants trigger settlement** — all consumed charges are aggregated into a single USDC transfer per cycle.

## Two Integration Modes

<Columns>
  <Column>
    **Mode A — Agon-Native**

    Embed the `@agonx402/platform` SDK on your API server. Payments are authorized entirely off-chain with full spending controls, budgets, and instant authorization.

    **Best for:** High-volume APIs, micropayment-heavy workloads, custom pricing logic.
  </Column>
  <Column>
    **Mode B — x402 Proxy**

    Keep your standard x402 server integration untouched. The Agon consumer SDK transparently proxies payments through Agon on the client side.

    **Best for:** Existing x402 merchants. *Note: The merchant must explicitly configure Agon's URL as their payment facilitator.*
  </Column>
</Columns>
