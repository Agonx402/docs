## What is x402?

HTTP 402 "Payment Required" is a dormant HTTP status code revived as an open protocol for machine-to-machine payments. When a client requests a paid resource, the server responds with `402` and a payment challenge. The client completes the payment and retries the request with proof.

The elegance of x402 is that payments are **protocol-level** — any HTTP client can pay any HTTP server without a shared SDK or integration agreement, as long as both speak the same payment standard.

The problem: every payment is an on-chain transaction. This works for infrequent, high-value API calls, but breaks down under high-frequency micropayment workloads.

## What is Agon?

Agon sits on top of x402 as a **payment aggregation layer**. It provides the off-chain authorization infrastructure that x402 is missing, enabling:

- **Instant authorization:** Sub-millisecond payment validation via off-chain ledger lookup
- **No per-call fees:** Thousands of calls collapse into one settlement transaction per merchant
- **Programmable spending:** Set per-request caps, daily limits, and multi-use payment tokens
- **Agent-native auth:** Headless registration and signing via Ed25519 keypair or agentic wallets

## Authorization vs. Settlement

These two concepts are the core of Agon's design:

<Columns>
  <Column>
    **Authorization (Off-chain)**

    A ledger operation that atomically moves funds from `available_balance` to `reserved_balance`. Happens in milliseconds. No blockchain involved.

    - Triggered by the merchant calling `POST /authorize`
    - Creates a **Reservation** — a time-limited hold (5 min TTL)
    - Instant: immune to RPC latency or network issues
  </Column>
  <Column>
    **Settlement (On-chain)**

    A real USDC transfer on Solana from Agon's operational wallet to each merchant's wallet. Triggered by the merchant via API or dashboard.

    - All charges to a merchant are **netted** into one sum
    - All merchant payouts are settled in **one Solana transaction**
    - The merchant receives real USDC — not an IOU
  </Column>
</Columns>

## Balances

Every Agon account carries two live balance fields:

| Field | Meaning |
|---|---|
| `available_balance` | Funds free to spend on API calls or withdraw on-chain |
| `reserved_balance` | Funds held for in-flight API calls — released automatically on failure |

> **Total balance** = `available_balance` + `reserved_balance`

<Note>
  Reservations auto-expire after **5 minutes** if neither consumed nor released. Funds always return to `available_balance`. You are never charged for a failed request.
</Note>

## Currency Units

All API amounts are in USDC **micro-units**. 1 USDC = 1,000,000 units.

| USD | Micro-units |
|---|---|
| $0.001 | 1,000 |
| $0.01 | 10,000 |
| $0.10 | 100,000 |
| $1.00 | 1,000,000 |
| $10.00 | 10,000,000 |

## The Custodial Model

Agon is a **custodial** payment aggregator. When you deposit USDC, Agon holds it in its operational wallet. Your balance is tracked in an off-chain ledger — the source of truth for all funds.

This is the same trust model as Stripe, PayPal, or a bank — you trust the intermediary to hold funds accurately and execute withdrawals faithfully.

<Warning>
  **Custodial reliance:** Agon is responsible for maintaining accurate balances. You can withdraw your full `available_balance` on-chain at any time with no lock-up.
</Warning>
